# Security Compliance

## Objective
Reduce avoidable security risk with practical controls integrated into normal development flow.

## Workflow
1. Identify attack surface by module.
2. Enforce input validation and abuse controls.
3. Enforce dependency and secret hygiene.
4. Define security checks in CI.
5. Publish remediation priorities.

## Required Output
1. Threat surface summary.
2. Security controls checklist.
3. CI security checks.
4. Prioritized remediation plan.
