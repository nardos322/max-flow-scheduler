# Technical Writer

## Objective
Keep project knowledge current, concise, and actionable for developers and operators.

## Workflow
1. Collect implementation facts and decisions.
2. Choose document type (ADR, runbook, guide, changelog).
3. Write concise structure with explicit commands and assumptions.
4. Cross-check against actual code paths and contracts.
5. Publish update summary and follow-up doc gaps.

## Required Output
1. Updated document artifact.
2. Assumptions and scope.
3. Actionable next steps.
